External, internal och inline CSS

Det finns tre olika sätt att addera sin CSS på. Inline innebär att man stylar ett eller flera separata element direkt i koden. Internal CSS innebär att man stylar en separat html-sida genom att addera en style-tagg i headen. I style-taggen lägger man alla de css-regler man vill applicera på den aktuella html-sidan. Det tredje sättet är att använda sig av externa stillmallar med CSS. Med en eller flera externa stilmallar kan man på ett enkelt sätt återanvända samma CSS upprepade gånger. Det blir också betydligt enklare att läsa koden på respektive html-sida när sidan inte innehåller både Html och CSS om vartannat.

Om man i stället har färre CSS-regler och ifall de är specifika för enbart en sida är det enklare att ha sin CSS samlad på den aktuella sidan. Jag ser inte många fördelar med att använda sig av inline CSS förutom när man har väldigt få element som ska stylas. I så fall är det enkelt och överskådligt att kunna se en koppling mellan attributet och dess formatering. Vanligtvis har man dock många html-sidor på en webbplats och då blir det oerhört praktiskt att ha externa stilmallar.
